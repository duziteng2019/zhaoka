# 证件照小程序 - 部署和测试指南

## 📋 部署前准备

### 1. 云函数依赖安装

每个云函数需要安装相应的依赖包。在微信开发者工具中：

```
云函数列表:
├── auth (依赖: wx-server-sdk)
├── upload (依赖: wx-server-sdk)
├── faceDetect (依赖: wx-server-sdk, tencentcloud-sdk-nodejs)
├── portraitSegment (依赖: wx-server-sdk, tencentcloud-sdk-nodejs)
├── cropImage (依赖: wx-server-sdk, sharp)
├── replaceBackground (依赖: wx-server-sdk, sharp)
├── createOrder (依赖: wx-server-sdk)
├── saveHistory (依赖: wx-server-sdk)
├── getHistory (依赖: wx-server-sdk)
└── deleteHistory (依赖: wx-server-sdk)
```

**安装步骤：**
1. 右键点击每个云函数文件夹
2. 选择 "在终端中打开"
3. 运行 `npm install`
4. 等待安装完成

**注意事项：**
- `sharp` 库需要本地编译，可能需要配置构建环境
- `tencentcloud-sdk-nodejs` 需要配置 API 密钥

### 2. 环境变量配置

需要在云开发控制台配置以下环境变量：

```
TENCENT_CLOUD_SECRET_ID=your_secret_id
TENCENT_CLOUD_SECRET_KEY=your_secret_key
```

### 3. 微信支付配置

**必须配置：**
- 微信支付商户号
- API 密钥
- 证书文件

**在 `createOrder` 云函数中配置：**
```javascript
const pay = new Wechatpay({
  appid: 'YOUR_APPID',
  mchid: 'YOUR_MCHID',
  publicKey: 'YOUR_PUBLIC_KEY',
  privateKey: 'YOUR_PRIVATE_KEY'
});
```

## 🚀 云函数部署步骤

### 批量部署方案

在微信开发者工具中：

1. **右键点击 `cloudfunctions` 文件夹**
2. 选择 "上传并部署：云端安装依赖"
3. 等待所有云函数部署完成

### 单个部署方案（推荐用于测试）

1. **按优先级逐个部署：**

```
第一阶段：基础服务
☐ auth - 认证服务
☐ upload - 图片上传

第二阶段：AI 处理
☐ faceDetect - 人脸检测
☐ portraitSegment - 人像分割
☐ cropImage - 图片裁剪
☐ replaceBackground - 背景替换

第三阶段：业务功能
☐ createOrder - 订单创建
☐ saveHistory - 保存历史
☐ getHistory - 获取历史
☐ deleteHistory - 删除历史
```

2. **每个云函数部署步骤：**
   - 右键点击云函数文件夹
   - 选择 "上传并部署：云端安装依赖（不上传 node_modules）"
   - 等待部署完成（约 30-60 秒）

## 🔧 常见问题排查

### 问题 1：云函数调用失败

**症状：**
```
Error: 云函数不存在
Error: 云函数调用超时
```

**排查步骤：**
1. 检查云函数是否已部署
2. 查看云函数日志
3. 验证云函数名称正确

**解决方案：**
- 重新部署云函数
- 增加云函数超时时间
- 检查网络连接

### 问题 2：图片上传失败

**症状：**
```
Error: 文件大小超限
Error: 云存储空间不足
```

**排查步骤：**
1. 检查图片大小（< 10MB）
2. 查看云存储配额
3. 验证文件格式

**解决方案：**
- 压缩图片后再上传
- 扩展云存储空间
- 使用支持图片格式

### 问题 3：AI API 调用失败

**症状：**
```
Error: 腾讯云 API 认证失败
Error: API 调用超时
```

**排查步骤：**
1. 验证 API 密钥配置
2. 检查 API 调用次数
3. 查看腾讯云控制台

**解决方案：**
- 配置正确的 SecretId 和 SecretKey
- 增加配额或充值
- 使用本地模型降本

### 问题 4：支付失败

**症状：**
```
Error: 支付参数错误
Error: 签名验证失败
```

**排查步骤：**
1. 检查微信支付配置
2. 验证订单金额
3. 查看支付日志

**解决方案：**
- 更新支付配置
- 使用正确的金额格式（分）
- 验证商户号权限
